# 信奥赛智能学习诊断系统 - 产品需求文档 (PRD)

## 1. 产品概述

信奥赛智能学习诊断系统是一款专为CSP-J/S信奥赛学生设计的纯前端学习辅助应用。系统通过AI驱动的水平诊断、个性化学习路径规划、智能题目推荐和错题归因分析，帮助学生科学备考，同时为家长提供可视化的进度报告，解决学生刷题盲目、家长无法辅导、老师时间不足的三方痛点。

**核心价值**：
- 为学生提供精准的学习诊断和个性化学习路径
- 为家长提供易懂的进度报告，让家长能够参与孩子的学习过程
- 纯前端实现，无需后端服务器，数据内置，演示效果好

## 2. 核心功能

### 2.1 功能模块

#### 2.1.1 核心页面
1. **首页**：系统介绍、学习数据概览、快速入口

3. **学习路径**：个性化学习路径展示、知识点进度追踪

## 4. 用户界面设计

### 4.1 设计风格

**整体风格**：现代科技感 + 教育亲和力
- **主色调**：深蓝色 (#1E3A8A) 代表专业与信任，辅以活力橙 (#F97316) 代表进步与成就
- **辅助色**：浅灰色背景 (#F3F4F6)，白色卡片 (#FFFFFF)，成功绿 (#10B981)，警告红 (#EF4444)
- **按钮风格**：圆角设计 (8px)，微阴影效果，hover时有轻微上浮动画
- **字体**：
  - 标题：思源黑体 / Source Han Sans CN (粗体)
  - 正文：思源黑体 / Source Han Sans CN (常规)
  - 代码：JetBrains Mono / Fira Code
- **布局风格**：卡片式布局，顶部导航栏，响应式设计
- **图标风格**：线性图标，统一线条粗细，配合微动画

### 4.3 响应式设计

- **桌面优先**：主要针对桌面端设计，宽度1200px-1920px
- **移动端适配**：响应式布局，支持平板和手机访问
- **触控优化**：移动端按钮和交互区域适当放大，支持手势操作

### 4.4 动画与交互

- **页面加载**：骨架屏加载，内容渐入动画
- **数据更新**：数字滚动动画，进度条填充动画
- **用户操作**：按钮点击涟漪效果，卡片hover阴影变化
- **成就反馈**：答题正确时显示庆祝动画，解锁知识点时显示解锁动画

## 5. 数据设计

### 5.1 内置数据结构

所有数据存储在 `src/data` 目录下的JSON/TS文件中：

```
src/data/
├── problems.json          # 题库数据
├── knowledgePoints.json   # 知识点数据
├── diagnosisQuestions.json # 诊断题目数据
├── learningPath.json      # 学习路径数据
├── userData.json          # 用户学习数据（localStorage同步）
└── aiResponses.json       # AI归因预设响应
```

### 5.2 核心数据模型

#### 题目数据
```typescript
interface Problem {
  id: string;
  title: string;
  difficulty: 'easy' | 'medium' | 'hard';
  knowledgePoints: string[];
  description: string;
  inputFormat: string;
  outputFormat: string;
  sampleInput: string;
  sampleOutput: string;
  testCases: TestCase[];
  hints: string[];
}

interface TestCase {
  input: string;
  output: string;
}
```

#### 知识点数据
```typescript
interface KnowledgePoint {
  id: string;
  name: string;
  category: string;
  description: string;
  prerequisites: string[];
  difficultyLevel: number;
  problems: string[];
}
```

#### 用户学习数据
```typescript
interface UserLearningData {
  diagnosisHistory: DiagnosisRecord[];
  learningProgress: KnowledgeProgress[];
  completedProblems: ProblemRecord[];
  mistakes: MistakeRecord[];
  studyTime: number;
  streakDays: number;
}

interface DiagnosisRecord {
  id: string;
  date: string;
  level: 'CSP-J' | 'CSP-S';
  scores: Record<string, number>;
  weakPoints: string[];
}

interface KnowledgeProgress {
  knowledgePointId: string;
  masteryLevel: number;
  completedProblems: number;
  correctRate: number;
}

interface MistakeRecord {
  problemId: string;
  date: string;
  code: string;
  errorType: string;
  aiAnalysis: string;
}
```
